# Security Policy

This project enables server-side persistence of MeshCommander
configuration data.

## Important

Saved computer entries may include credentials stored **unencrypted** on
disk.

## Recommendations

-   Protect the data directory with strict filesystem permissions
-   Avoid saving passwords if not required
-   Use HTTPS when exposing the service
-   Run on trusted infrastructure only

## Reporting Issues

If you believe you have found a security issue related to the container
or patch logic, please open a GitHub issue with details.

Do **not** include live credentials in issue reports.
